Menu Bar 快速切换 — 产品技术方案

状态: 设计评审中 版本: v0.1 日期: 2026-05-29

1. 背景

当前 MirrorKit 需要打开 APP 窗口才能切换镜像源。对于日常使用场景（如临时发现下载慢 -> 切换源 -> 继续下载），每次都要打开 APP 的流程过于繁琐。从 menu bar 直接操作可以大幅降低使用摩擦。

2. 功能目标

  1. 不打开 APP 窗口，从 mac 顶部菜单栏快速切换镜像源
  2. menu bar 图标显示当前镜像整体状态（正常/异常）
  3. 点击图标展开菜单，按工具分组，子菜单列出可选镜像源
  4. 切换后菜单自动关闭，无需额外确认
  5. 点击「打开 MirrorKit」可跳回主窗口

3. 整体架构

+-- App Process (always running) -----------------------------------+ | | | +-- NSApplication --------------------------------------------+ | | | ActivationPolicy: .accessory (no dock icon) | | | | | | | | +-- MirrorKitApp (SwiftUI App) --+ +-- MenuBarManager --+ | | | | | ContentView | | NSStatusBar | | | | | | NavigationSplitView | | NSMenu (动态构建) | | | | | | ... | | ... | | | | | +--------------------------------+ +--------------------+ | | | | | | | | +-- Services (shared by both) -------------------------+ | | | | | BrewMirrorService / NodeMirrorService / ... | | | | | | LatencyMeasurer / ShellProfileManager / ... | | | | | +------------------------------------------------------+ | | | +-------------------------------------------------------------+ | +-------------------------------------------------------------------+

3.1 核心设计：双模式共存

SwiftUI App 和 NSStatusBar 共享同一套 Service 层：

  • 主窗口（SwiftUI）— 现有的完整界面，无变化
  • Menu Bar（AppKit bridge）— 寄生在同一个 NSApplication 进程中
  • Service 层 — 两者共用，状态通过 UserDefaults + Notification 同步
进程模型：APP 启动后始终保持运行，关闭窗口时不让进程退出。用户可以从 menu bar 操作，也可以通过「打开 MirrorKit」菜单项唤回主窗口。

5. 实现步骤

5.1 AppKit bridge

在 SwiftUI 的 mirrorkitApp.swift 中通过 NSApplicationDelegateAdaptor 接入：

@main
struct MirrorKitApp: App {
    @NSApplicationDelegateAdaptor(AppDelegate.self) var appDelegate

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .windowResizability(.automatic)
    }
}

final class AppDelegate: NSObject, NSApplicationDelegate {
    private var menuBarManager: MenuBarManager?

    func applicationDidFinishLaunching(_ notification: Notification) {
        menuBarManager = MenuBarManager()
        NSApp.setActivationPolicy(.accessory)
    }
}

5.2 Service 协议抽象

为统一菜单构建，需要引入通用的 Service 协议：

protocol MirrorServiceProtocol {
    var displayName: String              // "Brew 镜像"
    var systemImage: String              // "mug"
    var mirrors: [any MirrorItem]        // 镜像源列表
    var activeMirrorId: String           // 当前选中
    var latencyResults: [String: TimeInterval]

    func applyMirror(id: String) async throws
    func startSpeedTest() async
}
这与 REFACTOR.md 中提出的「引入 MirrorProtocol」目标一致。如果已有协议抽象，menu bar 可以直接复用。

5.3 菜单构建

private func buildMenu() -> NSMenu {
    let menu = NSMenu()

    // 状态行
    menu.addItem(statusMenuItem())

    for service in services {
        let submenu = NSMenu()
        for mirror in service.mirrors {
            let item = NSMenuItem(
                title: mirror.name,
                action: #selector(switchMirror(_:)),
                representedObject: (service: service, mirrorId: mirror.id)
            )
            item.state = mirror.id == service.activeMirrorId ? .on : .off
            submenu.addItem(item)
        }
        let parent = NSMenuItem(title: service.displayName, submenu: submenu)
        parent.image = NSImage(systemSymbolName: service.systemImage, accessibilityDescription: nil)
        menu.addItem(parent)
    }

    menu.addItem(.separator())
    menu.addItem(NSMenuItem(title: "打开 MirrorKit", action: #selector(openWindow)))
    menu.addItem(NSMenuItem(title: "退出", action: #selector(quit)))
    return menu
}
