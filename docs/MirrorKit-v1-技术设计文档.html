01 概述与范围

1.1 产品定义

MirrorKit v1 是一款 macOS 桌面工具，专注于解决中国开发者在 Homebrew 使用中因网络限制导致的下载缓慢、连接失败问题。v1 仅做一件事——让 Homebrew 镜像源的管理和切换像点一下鼠标一样简单。

v1 核心范围：Homebrew 镜像源测速、一键切换、环境变量持久化。不包含代理管理、多工具 Registry 切换、DNS 诊断等功能（规划在后续版本）。

1.2 目标版本

平台macOS 15.6+（Sequoia）
技术栈Swift 6.0+ / SwiftUI + AppKit Hybrid
最低系统macOS 15.6+

设计语言Cask Noir · Prism（棱镜）子语言

3.2 测速引擎（SpeedTestService）

采用与 BrewCrate 一致的并行 HTTP 延迟测量方案，v1 增加测速结果缓存与历史对比。

测速流程

┌──────────┐    ┌───────────────┐    ┌─────────────┐
│ 用户点击  │───▶│ SpeedTest     │───▶│ URLSession  │
│ "测速"    │    │ Service       │    │ withTaskGroup│
└──────────┘    └───────┬───────┘    └──────┬──────┘
                        │                   │
                        ▼                   ▼
                 ┌──────────────┐   ┌──────────────┐
                 │ 并发请求所有   │   │ 每个源分别    │
                 │ 镜像源 bottle  │   │ 测量延迟      │
                 │ domain 根路径  │   │ (并行 7 路)   │
                 └──────┬───────┘   └──────┬──────┘
                        │                   │
                        ▼                   ▼
                 ┌──────────────────────────────────┐
                 │  结果聚合 → 排序 → 推荐最优       │
                 │  (按延迟升序，失败源排在末尾)       │
                 └──────────────────────────────────┘

接口设计

// SpeedTestService.swift
actor SpeedTestService {
    func measureAllSources() async -> [String: TimeInterval]
    // 对所有 preset 发起并发 HTTP HEAD/GET 请求，返回 { sourceId: latencyMs }

    func measureSource(_ source: MirrorSource) async -> TimeInterval?
    // 对单个源测量，返回毫秒延迟，失败返回 nil
}
测速实现要点
  • 目标 URL： 取 bottleDomain 根路径（如 https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/），官方源取 https://formulae.brew.sh/
  • 超时： 单次请求 10s 超时
  • 并发： withTaskGroup 并行 7 路，任一完成即记录
  • 缓存： 测速结果在 Store 层缓存，用户手动刷新或进入页面超过 60s 后自动刷新
  • 可视化： 以横向条状图显示相对延迟，最短的显示为 100% 宽度，其余按比例缩放

3.3 环境配置服务（ShellConfigService）

这是 MirrorKit 区别于 BrewCrate 的核心模块。BrewCrate 仅在启动子进程时注入环境变量，而 MirrorKit 需要将配置写入用户的 shell profile，使得全局终端会话都能生效。

管理的配置项

环境变量 / 配置值示例写入位置
HOMEBREW_BREW_GIT_REMOTEhttps://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git~/.zshrc / ~/.bash_profile
HOMEBREW_BOTTLE_DOMAINhttps://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles~/.zshrc / ~/.bash_profile
HOMEBREW_API_DOMAINhttps://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api~/.zshrc / ~/.bash_profile
git config --global url."<mirror>".insteadOfhttps://github.com.cnpmjs.org/ insteadOf https://github.com/~/.gitconfig

Shell 文件写入策略

// ShellConfigService.swift

/// MirrorKit 管理的 shell 配置区块标记
private let markerStart = "# --- MirrorKit Managed ---"
private let markerEnd   = "# --- End MirrorKit ---"

// 写入 ~/.zshrc
func applyMirrorConfig(_ source: MirrorSource) async throws {
    let envVars = MirrorEnvironmentBuilder.environmentVariables(mirror: source)
    // 构建区块内容：
    // # --- MirrorKit Managed ---
    // export HOMEBREW_BREW_GIT_REMOTE=...
    // export HOMEBREW_BOTTLE_DOMAIN=...
    // export HOMEBREW_API_DOMAIN=...
    // # --- End MirrorKit ---
    try writeBlockToShellProfile(content)
    try applyGitInsteadOf(source)
}

/// 恢复到官方源（清空区块）
func resetToOfficial() async throws {
    try removeBlockFromShellProfile()
    try removeGitInsteadOf()
}
注意：写入前自动备份原 ~/.zshrc（备份为 ~/.zshrc.mirrorkit.bak），写入操作通过文件读写实现，不依赖 sed 等外部命令，确保在 macOS 上一致的行为。切换失败时自动回滚备份。

3.4 镜像切换流程（MirrorService）

完整的"一键切换"编排由 MirrorService 完成，采用"备份 → 执行 → 提交 / 回滚"的事务式模式：

┌──────────────────────────────────────────────────────────────────┐
│  switchMirror(_ source: MirrorSource)                             │
├──────────────────────────────────────────────────────────────────┤
│  ① 备份当前 git remote 和 shell profile                           │
│  ② 设置 brew origin remote → source.brewGitRemote                │
│  ③ 写入 shell profile 区块（HOMEBREW_* 环境变量）                │
│  ④ 配置 git insteadOf 规则                                       │
│  ⑤ 全部成功 → 持久化选中状态到 UserDefaults                       │
│  ⑥ 任一步失败 → 回滚：恢复 remote / 恢复 shell / 恢复 gitconfig    │
│  ⑦ 返回 Result + 操作日志                        │
└──────────────────────────────────────────────────────────────────┘

接口声明

@MainActor
final class MirrorService {
    private let gitManager: GitRemoteManager
    private let shellConfig: ShellConfigService
    private let speedTest: SpeedTestService
    private let persistence: MirrorPreferenceStore

    var activeMirror: MirrorSource { persistence.selectedMirror }

    func selectMirror(_ source: MirrorSource) async throws
    func resetToOfficial() async throws
    func measureAll() async -> [String: TimeInterval]
}

5.3 设计语言

以 Cask Noir 为基础框架，MirrorKit 建立子语言 Prism（棱镜） —— 冷冽精密的光学感设计：

  • 深色主题（#0A0A0F），主强调色为 冷光青（#00F0FF），辅以琥珀金（#FFB800）作为操作反馈色
  • 边框即层级，不使用阴影；选中项以冷光青边框 + 内发光表示
  • SF Pro 界面字体 + SF Mono 等宽字体（日志/数据）
  • 测速条状图使用冷光青渐变：#00F0FF（低延迟）→ #007AFF（中延迟）→ #F87171（高延迟/失败）
  • 选中镜像源以冷光青高亮 + 镜面反射光晕效果
  • 测速时脉冲波纹动效（雷达扫描感），切换成功时镜面闪光动效

09 关键实现细节

9.1 Shell 区块读写

借鉴 BrewCrate 的 MirrorEnvironmentBuilder 生成环境变量键值对，但写入策略完全不同：

  • ShellConfigService 读取 ~/.zshrc（优先）或 ~/.bash_profile（fallback）全文
  • 查找 # --- MirrorKit Managed --- 标记，替换其与结束标记之间的全部内容
  • 如无标记，在文件末尾追加新区块
  • 写入前备份，写入后 source 命令不执行（仅在新建终端生效），通过菜单栏通知告知用户"打开新终端生效"
  • 官方源切换时，移除整个区块（而非写入空值）

9.2 Git insteadOf 规则管理

v1 仅管理 Homebrew 核心仓库的 remote URL，不涉及通用 GitHub insteadOf（后者规划在 v1.5 的多工具覆盖中）：

  • 切换镜像：git -C $(brew --repository) remote set-url origin <mirror-url>
  • 备份当前 origin URL 用于回滚
  • 切换官方源时恢复备份的 origin URL

9.3 地理位置检测

  • 启动时检测 Locale.current.region 是否为 "CN"
  • 是 → 没有已经配置过的镜像 → 弹出首次使用引导："检测到您在中国大陆，是否配置镜像源？"
  • 用户可勾选"不再提示"

9.4 错误处理与回滚

enum MirrorError: Error {
    case brewNotInstalled
    case brewRepositoryNotFound
    case gitOperationFailed(String)
    case shellProfileNotFound
    case shellProfileWriteFailed(String)
    case rollbackFailed(String)
    case unknown(String)
}

所有切换操作都遵循"备份 → 执行 → 提交/回滚"模式。任一步骤失败时，自动回滚所有已执行的操作并恢复备份。
