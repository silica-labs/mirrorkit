01 设计理念

Prism（棱镜） 是 Cask Noir 体系下为 MirrorKit 建立的子语言。以光学仪器为意象，传达精密、冷静、透明的产品气质——正如棱镜将光线分解为色谱，MirrorKit 将纷乱的网络配置解析为清晰的管控视图。

核心关键词：精密 · 冷冽 · 透明 · 光学 · 秩序
  • 精密： 每一个像素、间距、色彩都有精确的数值依据，不做模糊的设计决策
  • 冷冽： 暗色模式以深色底色 + 冷光青为主调；亮色模式以冷白底色 + 深蓝为主调。始终传递理性、专业、高效的情绪
  • 透明： 界面层级通过边框而非阴影表达，所有元素清晰可辨
  • 光学： 动效灵感来源于光的折射、反射和色散，非机械式的位移/淡入淡出
  • 秩序： 信息层级严格遵循网格系统，数据展示对齐精准

模式切换策略

  • 跟随系统设置（@Environment(\.colorScheme)），用户也可在设置面板中手动覆盖
  • 所有色彩值通过 CSS 自定义属性 / SwiftUI Environment 的 token 系统管理，一处定义全局生效
  • 切换时过渡动画 300ms ease，避免突兀闪烁
  • 强调色在两种模式下分别做对比度优化：暗色用冷光青 #00F0FF，亮色用深蓝 #0066FF，均满足 WCAG AA 标准

2.4 强调色切换逻辑

两模式使用不同的主强调色，这是 Prism 设计系统的关键决策：

  • 暗色模式（冷光青 #00F0FF）： 在深色底色上发光感强，传递科技感和冷静气质。与深色背景的对比度 ≥ 7:1
  • 亮色模式（深蓝 #0066FF）： 在浅色底色上清晰醒目，传递专业和可靠感。与浅色背景的对比度 ≥ 7:1
  • 两种强调色在视觉上保持一致的"冷色调"家族属性，确保品牌延续性
  • 辅强调色（Accent Warm）在亮色模式下从琥珀金 #FFB800 调整为深琥珀 #D97706，以保证在浅色背景上的可读性

  • 主内容区：自适应，最小 480px
  • 卡片网格：repeat(auto-fill, minmax(240px, 1fr))，gap 16px
  • 列表项高度：36px（紧凑）/ 44px（默认）

05 边框与层级

边框即层级，不使用阴影。通过边框的亮度差异区分元素的深度和交互状态。两模式均遵循此原则，仅颜色值不同：

层级暗色模式亮色模式
默认1px solid #1E1E2A1px solid #D1D1DB
悬停1px solid #2A2A3E1px solid #B8B8C8
选中1px solid #00F0FF + 内发光1px solid #0066FF + 内发光
行分割线1px solid #1E1E2A1px solid #D1D1DB
为什么不用阴影？阴影在暗色模式下难以感知且增加视觉噪点，在亮色模式下容易显得杂乱。用边框亮度区分层级更加清晰、可控，且符合"精密"的设计语言，两模式体验一致。

6.2 测速条 (LatencyBar)

  • 高度：8px，圆角：4px
  • 背景：暗色 #1E1E2A / 亮色 #D1D1DB
  • 填充：两模式均按延迟水平渐变（低延迟 → 中延迟 → 高延迟）
  • 测速中动效：脉冲扫描波纹（雷达感），600ms ease-out

6.3 状态指示器 (StatusIndicator)

两模式共用语义色，仅在使用 Surface 背景时微调透明度：

  • 🟢 正常：暗色 #34D399 / 亮色 #10B981
  • 🟡 警告：暗色 #FBBF24 / 亮色 #D97706
  • 🔴 错误：暗色 #F87171 / 亮色 #DC2626
  • ⚪ 未知/未检测：暗色 #5A5A6E / 亮色 #9A9AB0
  • 测速中：脉冲动画（透明度 0.4 → 1.0 循环，周期 800ms）

6.4 卡片

  • 边框圆角、内边距两模式一致（8px / 16px）
  • 选中态外发光颜色随模式切换：暗色冷光青 → 亮色深蓝

6.5 列表行

  • 高度、内边距两模式一致
  • 选中态左侧竖条颜色随模式切换：暗色冷光青 → 亮色深蓝

6.6 状态栏

  • 高度 28px，上边框分割，两模式一致
  • 分隔符 |：暗色 #3A3A4E / 亮色 #C8C8D4

07 动效规范

动效在两种模式下完全一致，不因模式不同而改变时序或曲线：

触发效果持续时间曲线
测速中LatencyBar 从右至左脉冲波纹扫描（雷达感）600msease-out
切换成功当前选中卡片镜面闪光（高光掠过）400msease-in-out
列表悬停行背景过渡到 accentDim150msease-out
状态变更StatusIndicator 颜色过渡300msease-in-out
面板切换内容区域十字淡入（cross-fade）200msease-out
通知出现从顶部滑入 + 轻微弹性300msspring(damping: 0.8)
进度/下载条状图填充从左到右推进800msease-out
模式切换全局背景/文字/边框色过渡300msease

动效原则

  1. 光学拟态： 动效灵感来源于光的物理行为，非机械式位移/淡入淡出
  2. 克制： 动效持续时间 200ms-600ms，不干扰操作效率
  3. 有意义： 每个动效都传递状态变化信息
  4. 模式无感： 动效不因亮暗模式而改变，确保交互体验一致

08 图标规范

  • 图标集： SF Symbols 6（Apple 原生图标库）
  • 尺寸： 16px（列表/行内）/ 20px（工具栏）/ 24px（空状态/大标题）
  • 字重： Regular（默认）/ Medium（选中/强调态）
  • 颜色： 默认 textSecondary，选中/强调 accent；颜色随模式自动切换
  • 层级符号： 侧边栏导航图标统一使用 16px Regular weight
  • 镜像源图标： 每个镜像源预设一个 SF Symbol + 20px 容器圆角 4px

09 双模式规范

9.1 模式切换触发

  • 默认： 跟随系统设置（通过 @Environment(\.colorScheme) 自动切换）
  • 手动覆盖： 设置面板中提供"亮色/暗色/跟随系统"三选项
  • 持久化： 用户选择存入 UserDefaults，下次启动时恢复

9.2 模式过渡动画

  • 切换时所有色彩相关的属性（背景、文字、边框、强调色）做 300ms ease 过渡
  • 利用 SwiftUI 的 .animation modifier 或 withAnimation 驱动，避免 DispatchQueue 手动控制
  • 任何视图组件不可硬编码颜色字面量，必须通过设计 token 间接引用，以确保自动跟随模式切换

9.3 模式测试清单

✅ 色彩

所有 token 在两模式下都有对应值，没有遗漏；对比度 ≥ WCAG AA

✅ 图片/图标

无硬编码深色背景的图片素材，使用 SF Symbols（自动适配）；如有自定义插图需提供 light/dark 两套

✅ 动效

模式切换本身有 300ms 过渡；其他动效不依赖模式

✅ 截图/导出

快照导出、日志导出等操作不应包含模式信息；如有截图功能需标注当前模式

11 设计 Token 映射

以下为 Swift 代码中对应的设计 token 定义参考，通过 Environment 或 @Environment(\.colorScheme) 实现自动切换：

// PrismColors.swift

extension Color {
    // Background
    static let prismBackground    = Color("prismBackground")    // #0A0A0F / #F5F5F7
    static let prismSurface       = Color("prismSurface")       // #12121A / #FFFFFF
    static let prismSurfaceAlt    = Color("prismSurfaceAlt")    // #18182A / #EBEBF0

    // Border
    static let prismBorder        = Color("prismBorder")        // #1E1E2A / #D1D1DB
    static let prismBorderHover   = Color("prismBorderHover")   // #2A2A3E / #B8B8C8

    // Text
    static let prismTextPrimary   = Color("prismTextPrimary")   // #E8E8ED / #1A1A2E
    static let prismTextSecondary = Color("prismTextSecondary") // #88889A / #6B6B80
    static let prismTextTertiary  = Color("prismTextTertiary")  // #5A5A6E / #9A9AB0

    // Accent
    static let prismAccent        = Color("prismAccent")        // #00F0FF / #0066FF
    static let prismAccentDim     = Color("prismAccentDim")     // 12% / 10%
    static let prismAccentWarm    = Color("prismAccentWarm")    // #FFB800 / #D97706

    // Semantic
    static let prismSuccess       = Color("prismSuccess")       // #34D399 / #10B981
    static let prismWarning       = Color("prismWarning")       // #FBBF24 / #D97706
    static let prismError         = Color("prismError")         // #F87171 / #DC2626
}
