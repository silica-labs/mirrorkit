01 官网定位与目标

1.1 官网使命

官网的首要目标是 驱动下载——让访客在 30 秒内理解 MirrorKit 解决什么问题，并产生下载试用的欲望。次要目标是建立品牌信任感，展示产品的专业性和精致度。

1.2 核心指标

指标目标衡量方式
页面停留时间> 45s网站分析
首屏可读性3 秒内传达产品价值五秒测试
下载转化率> 15% 访客点击下载CVR 追踪
信息传达用户能准确说出产品用途用户访谈
设计原则：少即是多。每个像素都服务于"让开发者下载试用"这个核心目标。

1.3 受众

  • 主要：中国开发者（中文内容，针对国内网络环境痛点）
  • 次要：对中国镜像源感兴趣的海外开发者（后续可做英文版）
  • 场景：在搜索引擎搜索 "brew 镜像"、"npm 镜像"、"github 加速" 时发现

1.4 竞品参考

方向参考对象可借鉴点
macOS 工具官网CleanMyMac, iStat Menus, Alfred大截图 + 特性卡片的信息层级
开发者工具官网Warp, Raycast, Figma暗色主题、终端/代码场景感
落地页结构Linear, Notion清晰的问题-解决方案叙事

03 视觉设计

3.1 设计语言

  • 风格：沉浸暗色、精致简约、macOS 原生质感
  • 关键词：Dark · Clean · Developer-oriented · Premium
  • 氛围：科技感、可靠、专注

3.2 色彩体系

与 App 内 Prism 设计系统保持一致，强化品牌一致性。

3.4 图标与图形

  • 品牌 Logo：从 Xcode Assets 导出的 AppIcon（1024x1024 png）
  • 工具图标：品牌色首字母方块（Monogram），每个工具对应一个特定色值
  • SVG 图标：核心操作用内联 SVG（下载箭头、GitHub 图标等），无需外部依赖
  • 特性卡片：不使用图标，以短横线 accent bar 作为视觉引领，靠排版建立层级
  • App 预览：真实应用截图 (screenshot.jpg)，展示 Dashboard 或镜像管理页
App 模拟 vs 真实截图考量：选择 CSS 模拟而非真实截图，因为模拟可以精确控制展示内容（展示理想的镜像测速状态），且加载无延迟、视觉风格完全可控。待产品稳定后可用真实截图替换。

4.4 核心特性（6 卡片）

每个卡片顶部有一条 accent 短横线作为视觉锚点，代替图标。靠标题字号 + 分隔线建立层级。

标题文案
一键切换内置镜像源列表，点选即可切换。无需记忆环境变量、git remote 等命令，告别繁琐的配置过程。
原生 macOS 体验纯 SwiftUI 构建，与 macOS 深度集成。精致的暗色主题，丝滑的交互动画，原汁原味的 Apple 设计语言。
实时测速自动检测各镜像延迟，数据可视化展示。帮助你快速选择当前网络条件下最快的镜像源。
安全可控所有操作在本地完成，镜像源配置写入 shell profile 或 git 配置。随时一键恢复官方源，无任何副作用。
多工具覆盖Brew、GitHub、Oh My Zsh、PyPI、Node.js、Rustup 等主流工具一站管理。统一的操作界面，零学习成本。
操作日志每次切换都有详细的操作日志，记录切换时间、源地址和执行结果。问题排查一目了然。

4.5 使用方式（三步）

CSS counter 自动编号 + 卡片式布局，编号为两位数字（01， 02, 03），带 accent 色。步骤之间用短横线 + 箭头连接。

步骤标题说明
01打开 MirrorKit启动应用，主界面清晰列出所有支持的工具
02选择工具点击任意工具进入镜像管理页，查看可用镜像源列表
03点击切换选择一个镜像源，应用自动完成所有配置，即刻生效

Footer：品牌标语 "以镜为鉴，可明得失" + 版权信息 + GitHub 链接。

维度定位示例

07 技术方案

7.1 技术选型

维度选择理由
语言纯 HTML + CSS无需构建工具，零依赖，GitHub Pages 可直接部署
CSS 管理内嵌在 HTML 的 <style> 中单文件部署，减少 HTTP 请求
图标CSS monogram + 内联 SVG零外部依赖，加载快，比 emoji 专业
App 预览CSS mockup精确控制展示内容，视觉一致
部署GitHub Pages免费，与项目同仓库
交互纯 CSS（hover、smooth scroll）无需 JavaScript

7.2 文件结构

mirrorkit-website/
├── index.html          ← 单页落地页（含全部 CSS）
└── assets/
    ├── icon.png        ← AppIcon 1024x1024
    ├── favicon.png     ← Favicon 64x64
    └── logo.png        ← Logo 783x561（备用）

7.3 性能目标

指标目标
页面大小< 100KB（不含图片）
HTTP 请求数≤ 4（HTML + 3 图片）
Lighthouse Performance> 95
Lighthouse Accessibility> 90

7.4 后续可迭代的技术增强

  • 暗色/亮色切换：导航栏按钮 + data-theme 属性 + localStorage 持久化。已实现，遵循系统偏好作为默认值。
  • 动效增强：Intersection Observer 驱动滚动渐入（当前使用 CSS transition）
  • 统计分析：集成 Plausible / Umami 等隐私友好的分析工具
  • 下载计数：对接 GitHub Releases API 展示下载量
