MirrorKit（镜箱）

产品需求文档 v0.1 — 面向中国开发者的本地开发网络环境治理工具
📅 2026-05-27 📌 状态：草案 Pre-Alpha

01 产品概述

MirrorKit（镜箱） 是一款面向中国开发者的 macOS 桌面工具（后续扩展至 Windows/Linux），专注于解决国内开发环境中因网络限制导致的各类连接、下载、同步问题。它通过统一管理镜像源、代理规则、DNS 解析和故障感知，让中国开发者的本地开发网络体验与海外开发者一样流畅。

一句话定位：一次配置，本地开发网络永不断联。

产品定位

MirrorKit 面向中国开发者本地开发中的网络治理需求，提供统一的镜像管理、代理联动、故障感知与环境快照能力——将碎片化的单点方案整合为完整的产品体验。

02 问题与机会

痛点分析

中国开发者在日常工作中面临的多层网络障碍：

层次问题影响频率
Homebrewbottle/api 镜像慢、版本落后brew install 耗时数十分钟每天
Gitclone 超时、push 失败代码协作受阻每天
Registrynpm/pip/go 等 registry 连接慢依赖安装失败每天
ReleaseGitHub Release 无法下载无法获取二进制/资源每周
Docker镜像拉取超时容器化开发受阻每周
DNS关键域名解析失败或慢所有网络操作受影响偶发

现有方案的不足

❌ 手工 alias/export

每个开发者维护一套 shell 配置，换机器就要重配，没有统一管理。

❌ 镜像站文档

分散在各镜像站页面，不同工具不同配置方式，每次都要搜索。

❌ 全局代理

Overkill — 有些场景不需要代理（如国内镜像），反而降低速度。

❌ 零散脚本

Raycast 插件、shell 脚本等，功能单一，没有联动和容灾。

产品机会

市场上缺少一个 专为中国开发者设计的、集镜像/代理/DNS/故障感知于一体的桌面产品。每个开发者都被这类问题消耗心智，但目前只能靠碎片化的知识拼凑方案。MirrorKit 的机会在于将这些散落的单点方案整合为统一的产品体验。

03 目标用户

用户画像特征核心需求
国内开发者（主力） 在中文网络环境下使用 macOS 进行软件开发 稳定的 git/brew/npm/pip 访问
出海团队国内研发 需要频繁访问 GitHub、海外 registry 可靠的海外资源获取通道
多设备开发者 在公司/个人多台 Mac 间切换 一键迁移网络配置
团队/企业 需要统一团队的开发环境网络配置 配置同步与标准化

04 核心功能

P0 — 必须（MVP）

4.1 智能测速与镜像切换

  • 内置中科大、清华、阿里云、华为云、官方等多个镜像源列表
  • 并发测速（HTTP 延迟 + 下载速度），自动推荐最优镜像
  • 一键切换 Homebrew 的 HOMEBREW_BOTTLE_DOMAIN、HOMEBREW_API_DOMAIN
  • 一键配置 Git insteadOf 规则（如 github.com → github.com.cnpmjs.org）
  • 支持 npm / pip / go / rustup / flutter 等常见工具的 registry 镜像切换

4.2 故障感知与自动容灾

  • 后台定时检测已配置的镜像可用性（ping + HTTP HEAD）
  • 镜像不可用时自动切换至备用镜像 + 系统通知
  • 镜像版本滞后检测（如 Homebrew formula 版本落后官方 7 天以上时告警）
  • 故障记录面板，展示历史故障时间线和恢复情况

4.3 环境快照

  • 一键导出当前所有镜像/代理配置为 .mirrorkit 文件
  • 一键从快照文件恢复配置
  • 快照包含：brew 镜像、git insteadOf、registry 配置、代理规则
  • 支持多快照管理（命名、删除、导出分享）

P1 — 重要（V1.0）

4.4 代理联动

  • 配置代理规则：哪些域名走代理、哪些走镜像、哪些直连
  • 支持 HTTP/HTTPS/SOCKS5 代理
  • 和镜像配置联动：镜像覆盖不了的域名自动走代理
  • 代理可用性检测（连接测试）

4.5 GitHub Release 加速下载

  • 检测到 curl/wget 下载 GitHub Release 时自动替换 URL 为国内镜像
  • 支持自定义加速规则（URL 匹配模式）
  • 下载队列与断点续传

4.6 DNS 诊断与优化

  • 一键检测关键域名（github.com、raw.githubusercontent.com 等）的 DNS 解析结果和延迟
  • 提供 DNS 切换建议（系统 DNS / 公共 DNS / 镜像站 DNS）
  • 支持自定义 hosts 管理

P2 — 增强（后续版本）

4.7 多设备同步

  • 登录账户后多设备间自动同步配置
  • 团队配置模板分享

4.8 诊断助手

  • 当 git clone / brew install 失败时，一键诊断网络原因
  • 输出诊断报告：DNS → 连通性 → 镜像状态 → 代理状态链路分析

4.9 命令行界面（CLI）

  • mirrorkit status — 当前镜像/代理状态概览
  • mirrorkit switch <source> — 一键切换镜像
  • mirrorkit snapshot [export|import] — 快照管理
  • CLI 和 GUI 共享同一套配置后端

关键设计决策

决策方案理由
配置持久化Shell config 片段写入 ~/.zshrc/~/.gitconfig 等原生位置不锁入自家格式，用户删除 App 后配置仍然生效
镜像源列表JSON 配置文件，内置常用源，支持用户自定义灵活，镜像站地址变更时无需发版
后台检测LaunchAgent 轻量 daemon，定时执行检测任务用系统原生机制，不驻留无用进程
CLI 架构Swift Package 产物，与 GUI 共享 Service 层代码避免重复实现，CLI 和 GUI 行为一致

08 里程碑与路线图

阶段目标核心功能预期
POC / CLI 原型 验证核心价值，获取社区反馈 CLI: mirrorkit switch <source>、mirrorkit speedtest、mirrorkit status 2 周
MVP (v0.1) 覆盖 Homebrew 镜像管理核心场景 GUI: 镜像列表 + 测速 + 一键切换、故障感知 + 通知、环境快照导出/导入 CLI 确认后 4 周
V1.0 多工具覆盖、稳定可用 Git/npm/pip/go 镜像、代理联动、GitHub Release 加速、DNS 诊断 MVP 后 6 周
V1.5+ 多设备、团队协作 账户系统、多设备同步、团队模板、CLI 正式版 V1.0 后 8 周
关键验证节点：CLI 原型发布后，观察 GitHub 下载量和 Issue 反馈。如果 2 周内 GitHub Star < 50 或 Issue 活跃度低，应重新评估需求真实性。

09 商业模式

层定价功能范围
Free 免费 核心镜像切换 + 测速 + 故障通知 + 环境快照
Pro 一次性购买 / 订阅 多设备同步 + 团队模板 + 代理联动 + GitHub Release 加速 + 高级诊断

Free 层解决"我的网络为什么这么慢"的认知问题，Pro 解决多设备/团队的一致性问题。CLI 工具保持开源免费，作为引流入口。

10 风险与应对

风险等级应对
用户不愿意为"本来能免费搞定"的事付费 高 先用开源 CLI 验证真实需求；Pro 功能针对多设备/团队场景，个人使用免费
镜像站地址变更或下线导致产品失效 中 镜像列表 JSON 配置化，云端的更新机制；内置fallback到官方源
网络限制政策变化 中 架构上保持灵活，以规则引擎为核心，政策变化时只需更新规则
竞品（如 Raycast 插件、Terminal 脚本）抢先覆盖 低 MirrorKit 的核心价值不是功能多，而是统一体验 + 容灾 + 多设备——碎片化工具做不了的
产品定位不够聚焦 中 严守"网络治理"边界，不膨胀进入包管理、系统优化等领域
